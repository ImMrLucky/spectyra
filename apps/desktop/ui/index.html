Setup Guide

How to connect your LLM tool

Any app that lets you change the API base URL can use Spectyra. The companion server acts as a local middleman — it optimizes your messages, then forwards them directly to the provider.

  1. Add your provider API key on the Provider Keys page (or set it as an environment variable like OPENAI_API_KEY)
  2. In your LLM app's settings, find the option for "API Base URL", "Custom Endpoint", or similar
  3. Change it to: http://127.0.0.1:4111/v1
  4. Keep your provider API key in the app as normal — Spectyra passes it through
  5. Run a task and check the Runs page to see savings

Examples

Environment variable:

export OPENAI_BASE_URL=http://127.0.0.1:4111/v1

OpenAI Python SDK:

client = OpenAI(base_url="http://127.0.0.1:4111/v1")

OpenAI Node SDK:

const openai = new OpenAI({ baseURL: "http://127.0.0.1:4111/v1" });

Security & Data Handling

Inference Path

Direct to provider. Your prompts and responses go straight from this machine to your LLM provider. Spectyra servers are never in the path.

Provider Billing

Your provider account, your API key. Spectyra does not pay for or see your provider usage.

Telemetry

Local only by default. Aggregated, redacted analytics can optionally be synced to the Spectyra cloud dashboard. Raw prompts are never uploaded.

Prompt Storage

Before/after prompt snapshots are stored locally in ~/.spectyra/companion/. They are never uploaded unless you explicitly opt in.

Cloud Relay

None. No live inference traffic is routed through Spectyra servers. The companion server binds to 127.0.0.1 only.

Data Location

All data is stored at: ~/.spectyra/

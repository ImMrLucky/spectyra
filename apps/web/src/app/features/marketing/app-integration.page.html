In-app SDK

For developers, backend services, internal apps, and teams calling LLM APIs from Node — add Spectyra in a few lines, wrap the calls you already make, and trim cost without changing the outcome you intend.

Install @spectyra/sdk, create one createSpectyra() instance, pass your existing OpenAI / Anthropic / Groq client into complete(). Optimization runs in your process; the provider SDK still calls the model with your API key. No Spectyra proxy for inference.

View savings while you build

In development, the fastest loop is to log fields from the object returned by complete() (see Local debug & analytics below). No extra services required.

For a richer live view in the browser, you can optionally run Spectyra’s in-app local companion alongside your app. That is not required for production apps that only embed the SDK — it is an optional aid when you want a localhost dashboard during development.

Optional: local browser dashboard

The @spectyra/local-companion package powers the same style of local HTTP + browser UI used in OpenClaw flows, but you can run it on its own for local experimentation. Think of it as a separate optional surface for localhost visibility — not part of the minimal SDK install path.

Typical dev workflow: run your app locally → (optionally) start the local companion → open the dashboard URL it prints → watch runs that go through the companion’s HTTP surface during that session. Embedded spectyra.complete() traffic in your own Node process is still best inspected via logs or normalized events unless you wire a custom bridge.

How the pieces relate

  • SDK wraps your LLM call and produces savings metadata (and optional normalized events) in-process.
  • Optional local companion serves a small local dashboard for traffic that hits its routes during dev.
  • Production does not depend on the browser dashboard — ship the SDK in your service as usual.

Local debug & analytics

Optional depth below the quick start — the main objects returned or emitted by the SDK:

  • providerResult — raw vendor response; use to verify behavior matches an unwrapped call.
  • report — primary savings object: tokens in/out, estimated costs, savings %, transforms, hints, provider/model, timestamps (see next section).
  • promptComparison — when prompt snapshots are enabled, includes diffSummary (tokens before/after, % saved, transforms) without sending raw prompt text to Spectyra by default.
  • flowSignals — pipeline signals (e.g. stability / stuck-loop style hints) for agentic or chat-heavy flows.
  • startSpectyraSession(...) — multi-step workflows: each session.complete() records a step; session.finish() returns aggregated session analytics.
  • sdkEventEngine / moatPhase34SummariesFromSdkBuffer() — advanced: normalized events and local summaries for deeper tooling (see SDK README).

What report is good for

  • Input / output token counts and estimated cost before vs after optimization
  • Estimated savings and savings percentage; transforms applied
  • Context, duplicate, and flow reduction percentages when present
  • Repeated-context / repeated tool-output hints; message turn counts
  • Provider, model, and timestamps — useful for correlating with your own logs

Default paths keep raw prompts and provider secrets in your environment. Cloud telemetry and session sync only send aggregated / redacted payloads you configure (e.g. API key + POST /v1/telemetry/run, or optional session sync) — not your OpenAI key and not full prompt bodies unless you explicitly opt into higher-risk modes documented in the SDK.

Production & aggregate analytics

During development: use console.log, the objects above, or the optional local dashboard for detail.

Set telemetry: { mode: "cloud_redacted" } so each complete() POSTs aggregated usage plus a bounded diagnostics object (no prompt bodies). SPECTYRA_LICENSE_KEY — where it comes from

A Spectyra license key (not your OpenAI key) unlocks full optimization when your org is entitled. Generate it in the web app: Plan & Billing → Desktop / Companion license keys → Generate license key (copy once). Set export SPECTYRA_LICENSE_KEY='…' or use your secret manager.

For cloud dashboards without a license file on disk, use a Spectyra API key plus spectyraCloudApiKey / spectyraApiBaseUrl per the SDK README — separate from license keys; use what fits your deployment.

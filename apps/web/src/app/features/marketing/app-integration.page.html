In-app integration

Add @spectyra/sdk to your backend or service. Spectyra runs the same local optimization stack as the OpenClaw Local Companion: your OpenAI-compatible messages are analyzed and trimmed in-process, then your existing SDK client calls the provider with your API key. Nothing is proxied through Spectyra for inference.

Add to your code (three steps)

  1. Create one createSpectyra({ … }) instance (reuse it; optional licenseKey, runMode, learning profile, workflow policy).
  2. Wrap each LLM call with spectyra.complete({ provider, client, model, messages }, adapter) using the adapter for your vendor.
  3. Read providerResult for the normal model response and report / promptComparison for savings and optional before/after snapshots.

What it does

  • Before the provider call: feature detection + the full optimization pipeline (whitespace, dedup, context shaping, tool-safe merges, etc.)—aligned with Local Companion behavior, including OpenAI tool threads (optimized text merged per row while preserving tool_calls).
  • Run modes: on applies changes when licensed; observe keeps the provider on full prompts but fills the report with projected savings; off skips optimization for licensed debugging.
  • License: without a valid key, the pipeline still runs in preview mode so you can show “what you’d save” (report.projectedSavingsIfActivated / limited transforms in analytics) while the provider receives the original messages.

How to see savings

  • Per request: inspect report.inputTokensBefore, report.inputTokensAfter, report.estimatedSavings / estimatedSavingsPct, and report.transformsApplied.
  • Explainability: report.notes (e.g. observe mode, tool-thread merge), report.duplicateReductionPct, flow hints, and repeated-context estimates mirror the companion-style report.
  • Prompt snapshots: when promptSnapshots is not none, use promptComparison.diffSummary for tokens saved and transform ids.
  • Aggregate / dashboards: emit normalized events (same event model as companion) and use helpers such as moatPhase34SummariesFromSdkBuffer() for execution-graph / state-delta style summaries—see the SDK README.

SPECTYRA_LICENSE_KEY — where it comes from

The license key is a long-lived Spectyra credential (not your OpenAI key). It unlocks full optimization when your org is entitled, and is used by the SDK, Local Companion, and Desktop for offline-friendly checks.

  1. Sign in to the Spectyra web app with the account that owns your workspace.
  2. Open Plan & Billing → scroll to Desktop / Companion license keys.
  3. Click Generate license key, give the device or service a label, and copy the key once — it is not shown again. Set it in your environment, e.g. export SPECTYRA_LICENSE_KEY='spectyra_…' (or your host’s secret manager).

Cloud dashboards only: for per-call usage in the web app without a license file, you can use a Spectyra API key and spectyraCloudApiKey / spectyraApiBaseUrl instead — see the SDK README (“telemetry”). License keys and API keys are separate; you may use one or both depending on your deployment.

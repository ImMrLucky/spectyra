OpenClaw integration

Use Spectyra with OpenClaw and the local companion — optimize token usage on your machine and send requests directly to your AI provider with your own keys. This path does not require the Spectyra Desktop app.

Other ways to integrate — We are working on additional integration types (for example embedding the SDK in your own applications). Those are not ready yet; this page focuses on OpenClaw + local companion only.

Who this is for

  • You already use OpenClaw (or plan to) as your AI assistant / agent runtime.
  • You want spectyra/* models to route through Spectyra on http://127.0.0.1:4111/v1 with optimization turned on.
  • You can install the Spectyra Local Companion via npm and start it — no Spectyra account is required for local optimization and the dashboard.

1. Install OpenClaw

If OpenClaw is not installed yet, follow the official install steps for your OS. The OpenClaw CLI is what you use to install skills from ClawHub and talk to your local gateway.

OpenClaw documentation

2. Install the Spectyra skill from ClawHub

The Spectyra skill is published for OpenClaw and lists install steps (including the ClawHub / CLI flow). After OpenClaw is on your PATH, install the skill — for example: 

openclaw skills install spectyra

Browse the public skill registry and learn how skills are versioned and installed: ClawHub · ClawHub in the OpenClaw docs

3. Install & run the Local Companion

Once you have the Spectyra OpenClaw skill, install the local companion to see the savings in real time dashboard in your browser. No Spectyra account is required for this path. 

npm install -g @spectyra/local-companion@latest && spectyra-companion start --open

Dashboard: http://127.0.0.1:4111/dashboard — token and cost savings update as you use OpenClaw with spectyra/* models.

OpenClaw provider config (reference info)

OpenClaw should call Spectyra’s OpenAI-compatible endpoint. Your skill / docs may merge this for you; this JSON is the usual models.providers.spectyra shape: 

{{ openclawConfigJson }}

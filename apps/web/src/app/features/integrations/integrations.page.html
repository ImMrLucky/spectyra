OpenClaw — How it works

What it is

Spectyra runs locally and acts as a local OpenAI-compatible model provider for OpenClaw (and similar tools).

Data flow

OpenClaw → Spectyra Local Companion on localhost → your selected real provider (using your API key).

What stays local

Prompts, responses, prompt comparisons, and analytics by default.

What leaves your machine

Only the final request to your chosen provider. Optional redacted analytics if you enable cloud sync.

Model aliases

spectyra/smart and spectyra/fast are not separate vendors — they are stable Spectyra-local routing profiles. Spectyra forwards to the real model you configured in Desktop (for example Claude Sonnet, GPT-4.1 mini, or a Groq fast model).

OpenClaw config example

Add models.providers.spectyra pointing at the Local Companion base URL. OpenClaw model refs use provider/model (e.g. spectyra/smart).

Other agentic coding tools

Any tool that supports a local OpenAI-compatible base URL can use the same Local Companion endpoint (http://127.0.0.1:4111/v1 by default). No OpenClaw-specific logic lives in Spectyra’s optimizer — only the documented OpenAI-compatible surface and stable spectyra/* aliases.
